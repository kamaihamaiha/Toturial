## 设计模式
- 单例模式（singleton）
- 简单工厂模式（SimpleFactoryPattern）
- 工厂方法模式（Factory Method）
- 抽象工厂模式（Abstract Factory）
- 策略模式（Strategy Pattern）
- 适配器模式（Adapter Pattern）
- 代理模式（Proxy Pattern）
- 模板方法模式（Template Method）
- 建造者模式（Builder Pattern）
- 外观模式（Facade Pattern）

### 1. 定义
- 某类特定问题的代码设计解决方案
- 是一套代码设计的经验总结

### 2. 作用
- 提高代码复用性，降低开发成本和周期
- 提高代码可维护性、可扩展性
- 是代码更优雅
- 让代码更容易让他人理解

### 3. 设计模式的设计原则
- 单一职责原则
- 开放封闭原则
- 里氏代替原则
- 依赖倒置原则
- 接口隔离原则
- 合成复用原则
- 最小知识原则
